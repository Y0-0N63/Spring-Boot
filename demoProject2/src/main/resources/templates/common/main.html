1. 템플릿 엔진

템플릿 양식과 특정 데이터 모델에 따른 입력 자료를 합성하여 결과 문서(응답 화면)를 출력하는 것 (View 계층)
-> 만들어둔 화면(HTML)에 데이터를 추가해 최종적인 HTML 문서를 생성하고 웹 브라우저에 전달(클라이언트에게 응답) (JSP도 템플릿 엔진!)
  타임리프 홈페이지

2. Thymeleaf

웹 및 독립 실행형 환경을 모두 지원하는 최신 서버 측 Java 템플릿 엔진
-> 웹 실행 = 요청 시 포워드 되는 화면, 컨트롤러를 통해 전달받은 동적 데이터를 HTML 템플릿에 채워넣어 > 최종적인 웹 페이지 제작 (일반적인 웹 서비스 동작) 
-> 독립 실행(HTML 파일을 웹 브라우저에서 직접 여는 환경)  = HTML 파일만 실행, 순수한 HTML로만 렌더링
	= Natural 템플릿 엔진 (서버 사이드 랜더링을 거치지 않고, 브라우저에 바로 띄워도 정상적인 화면을 볼 수 있음)
HTML 파일에서 th(Thymeleaf) 속성을 이용해 컨트롤러로 부터 전달 받은 데이터를 이용해 동적 페이지를 만들 수 있음
Spring Boot에서는 JSP가 아닌 Thymeleaf 사용을 권장하고 있음
  > Spring Boot에서는 기본적으로 JSP 컴파일러를 지원해주지 않아 tomcat-embed-jasper 의존성을 추가해야 함
